# 整理一下设计模式

# [pipeline](pipeline)

责任链模式,v1是一个简单的实现,v2是一个复杂的实现.

# [proxy](proxy)

代理模式,v1静态代理(接口代理,子类代理),v2动态代理.

# [singleton](singleton)

单例

# [decorator](decorator)

装饰器模式和代理模式很像，但是代理类和被代理类功能无关，而装饰器类是对原类功能的增强。
这装饰器类只需要实现它需要增强的方法就可以了，其他方法继承装饰器父类的默认实现。

# [prototype](prototype)

原型模式,深浅拷贝

# [composite](composite)

组合模式
